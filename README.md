# Invoice Creator

A small single-page web app for generating monthly consulting invoices as PDFs
directly from the browser. Fill in an invoice form, preview it live, and use the
browser's **Print → Save as PDF** to produce the final document — no server-side
PDF rendering required.

Invoices are saved per user in the cloud, so the same set of invoices is
available from any device after signing in. It's tailored for international
invoicing (USD amounts, SWIFT/IBAN beneficiary and intermediary bank details).

## Features

- Create, duplicate, edit, and delete invoices
- Live invoice preview that doubles as the print/PDF layout
- Per-user data isolation (you only ever see your own invoices)
- Email sign-in / sign-up via a hosted authenticator UI
- Fields for both direct and intermediary (correspondent) bank routing

## Tech stack

**Frontend**
- [React 18](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- [Vite 5](https://vite.dev/) for dev server and production builds
- [AWS Amplify UI React](https://ui.docs.amplify.aws/) (`@aws-amplify/ui-react`)
  for the authentication UI
- Plain CSS (`src/styles.css`), including a print stylesheet for PDF output

**Backend (Amplify Gen 2)**
- [AWS Amplify](https://docs.amplify.aws/) Gen 2 backend defined as code in
  `amplify/` (`aws-amplify` + `@aws-amplify/backend`)
- Backend resources are declared in TypeScript:
  - `amplify/auth/resource.ts` — authentication (email login)
  - `amplify/data/resource.ts` — the `Invoice` data model + authorization rules

## AWS services

The backend is provisioned by Amplify Gen 2 and maps to these AWS products:

- **AWS Amplify Hosting** — builds and serves the Vite frontend, and runs the
  backend pipeline deploy on each push (see `amplify.yml`).
- **Amazon Cognito** — user pools and identity pools for email-based
  sign-in/sign-up (`amplify/auth/resource.ts`).
- **AWS AppSync** — GraphQL API generated from the `Invoice` schema.
- **Amazon DynamoDB** — backing store for invoice records.

Owner-based authorization (`allow.owner()`) ensures each authenticated user can
only read and write their own invoices.

## Deployment

Deployment is fully managed by **AWS Amplify Hosting**, driven by the build spec
in [`amplify.yml`](./amplify.yml):

1. On every push to a connected branch, Amplify runs the **backend** phase,
   which deploys the Amplify Gen 2 stack via
   `npx ampx pipeline-deploy --branch $AWS_BRANCH --app-id $AWS_APP_ID`.
2. It then runs the **frontend** phase (`npm run build`) and publishes the
   `dist/` artifacts to the Amplify CDN.

Backend configuration consumed by the frontend at runtime lives in
`amplify_outputs.json`, which is generated by Amplify.

## Local development

Prerequisites: Node.js and npm.

```bash
npm install
npm run dev      # start the Vite dev server
```

Other scripts:

```bash
npm run build    # type-check (tsc -b) and build for production
npm run preview  # preview the production build locally
```

To run against a personal cloud sandbox backend (Amplify Gen 2):

```bash
npx ampx sandbox
```

This provisions a temporary backend and writes a local `amplify_outputs.json`
that the app reads on startup.

## Project structure

```
amplify/            Amplify Gen 2 backend (auth + data definitions)
  auth/resource.ts  Cognito email authentication
  data/resource.ts  Invoice model + owner authorization
  backend.ts        Backend composition
src/                React frontend
  App.tsx           App shell + auth gate + print handler
  InvoiceForm.tsx   Editable invoice fields
  InvoiceList.tsx   Sidebar list of saved invoices
  InvoicePreview.tsx Print/PDF layout
  useInvoices.ts    Data hooks (CRUD via the Amplify data client)
amplify.yml         Amplify Hosting build spec
```
